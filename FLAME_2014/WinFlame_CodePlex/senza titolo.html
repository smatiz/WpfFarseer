Flame is a library and an IDE whose aim is to improve and make easier to write code with IronRuby, IronPython, IronJS, Powershell, C# and potentially any .Net language.
I created Flame ..for..

Flame can be used :
  • for creating simple gui with scripting languages
  • for simple iteractive drawing
  • for making LINQ queries
  • for quickly testing or learning a new .NET assembly
  • for adding scripting enhancement for the users of your own application
  • as a flexible debug and test system for your own application
  • as a calculator, you can do iteractive math and even plots
  • or a mix of some of this!
It has the following advantages:
  • you can use different languages
  • you have access to all .net assemblies
  • sintax highlighting, autocompletition and shortcut

It is easy to start playing with Flame:

Download the latest release.
Link reference to dlls from .\WinFlame\DLLS\ directory into your C# WindowFormApplication project.
Write and execute somewhere in your code
[
    System.Windows.Forms.Form f = Flame.Controls.ScripterControlForm.ShowForm(".\\one.bin", new Flame.Dlr.CompilersManager.Parameters())
    f.Show();
]

Now you get a window and you can start coding in IronRuby:



Write [ "Hello world" ] into the first cell and press Shift + Enter.



Let's give some explanations.

Flame.Controls.ScripterControlForm.ShowForm is a static function that return a Flame Window Form. We create this static function just for simplifying first approach to Flame, and as an example of use.
The first argument is a string: the path to a file that will be used for saving and loading your code. The second argument is a class we will fill later to give more flexibility and to extend Flame.

Now write[System::Windows::Forms::MessageBox.Show("Hello world")] and press Shift + Enter.



as we can see, when we execute the code, a MessageBox appears, and when we close MessageBox the output in the comand window is [OK].
Behind the scenes Flame takes the output of [System::Windows::Forms::MessageBox.Show("Hello world")], and gives you back the object (of type System.Windows.Forms.DialogResult) as a String.
We will see later that this is not exactly true.

Too easy, let's make some math!
Try [ a = 7 - 5 ] and [b = System::Math.Sqrt(a)] Shift + Enter.



The output of Flame is always given by the output of last line of code.
At this point we should understand that the output is always a string, right? Wrong!
The output is always a System::Windows::Forms::Control (or a list of Control)!

NOTE: When it requires to show a string, actually, it shows a System::Windows::Forms::Label which Text property is equal to the required string.

Try
[
    b = System::Windows::Forms::Button.new()
    b.Text = "Hello World"
    b
]
and add an event to the button
[
    b.Click do |s, o| System::Windows::Forms::MessageBox.Show("my first event handler") end
]


Click the button!
